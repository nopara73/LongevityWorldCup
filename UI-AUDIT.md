# Longevity World Cup UI/UX Audit

Scope: no-code audit of the production and local screenshot baseline in `.artifacts/screenshot-baseline/`, interpreted through `ART-DIRECTION.md` and `DESIGN.md`.

The product direction is clear: Longevity World Cup should feel like the official scoreboard for age reversal. The local direction is substantially closer to that identity than production, especially on the homepage hero, CTA color system, condensed display type, and dark-scoreboard-to-light-paper page structure. The remaining problems are mostly about responsive execution, trust, sparse states, and places where production-era generic web UI still leaks through.

## Overall Read

| Area | Score | Notes |
|---|---:|---|
| Brand distinctiveness | 3/4 | The local version has a memorable scoreboard identity. Production still carries cyan/pink and generic SaaS cues. |
| Responsive UX | 2/4 | Core content works on mobile, but headers, drawers, long feeds, and task flows need tighter small-screen hierarchy. |
| Trust and evidence | 2/4 | Athlete data is public and compelling, but proof/evidence, sparse states, and media downloads do not always look deliberate. |
| Data usability | 3/4 | Dense leaderboard tables are a strength. Sparse ranking views and event history need better framing. |
| Visual system consistency | 2/4 | Local has the right palette and type direction, but utility pages still need restraint and production still has off-brand accents. |

## Preserve

- Keep the production information architecture: hero, podium, leaderboard, event/history surfaces, rules/about/history docs, media kit, and participation flows are the right product skeleton.
- Preserve local's dark scoreboard hero, green score language, amber crowd/amateur accent use, Anton-style competitive headings, and light paper content surfaces.
- Keep dense leaderboard tables. Density is a brand asset here because the product is a public competition, not a generic marketing site.
- Keep human faces, athlete modals, proof concepts, public events, and rules prominent. They are the trust system.

## Highest-Leverage Next Design Pass

1. Fix the global sticky/header overlap and mobile filter drawer first; these affect many routes and immediately improve perceived quality.
2. Make trust surfaces explicit: athlete proof access, sparse leaderboard explanations, and media-kit previews.
3. Bring remaining production-era pages fully onto the local brand system without making task pages overly poster-like.
4. Improve mobile task flows by reducing chrome and moving the primary input/action earlier.
